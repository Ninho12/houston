Como contribuir

Veja como você pode contribuir para fazer o Houston ficar ainda melhor!

Repositório Monorepo#

Utilizamos uma arquitetura monorepo para facilitar na hora de desenvolver, mas tentamos manter de uma maneira simples para que fique fácil de entender.

Atenção#

  • Não utilize o npm pois utilizamos o workspaces do yarn para ele instalar todas as depedências de todos os projetos;
  • Todas as funcionalidades Web devem estar no projeto dev, com o máximo de exemplos e codígos de exemplo possível;
  • Lembre-se que o Houston™️ também será usado com o react-native, menos o package de ui-components;
  • Todos os componentes previstos estão no Figma.

Arquitetura#

No projeto principal (@eduzz/houston-ui) discutimos muito sobre o uso de uma lib base para não termos que fazer tudo do zero e ganhar um pouco de velocidade no inicio, ponderamos sobre o AntD mas chegamos a conclusão que ele erá muito pesado e mudava os padrões que já aplicamos em projeto atuais, como quase todos os produtos já utilizam o MaterialUI, optamos por continuar usando ele como base, mas de uma maneira transparente para caso no futuro decidirmos remover ele, não terá impacto na maneira como usamos os componentes.

Para não impactar uma aplicação que já usa o sistema de temas do MaterialUI, criamos o WrapperTheme que aplicará o tema do Houston™ apenas no componentes do Houston™.

Como rodar#

  1. Clone o repositório do GitHub
git clone https://github.com/eduzz/houston.git
  1. Instale as depedências do projeto com o yarn
yarn
  1. Inicialize o projeto
yarn start
  1. Abra no VSCode o projeto pelo main.code-workspace
  2. Instale as extensões recomendadas

Ele irá construir as aplicações e iniciar todos os projetos, além de subir o projeto da documentação no http://localhost:3000.

Como iniciar uma tarefa#

  1. Veja o quadro de tarefas na aba Projects no GitHub;
  2. Escolha uma tarefa a seu gosto;
  3. Mova para a coluna de RFC (Request for Comments). Na issue da tarefa adicione a sua proposta de como o componente deve se comportar e, automaticamente, será enviado uma mensagem no Discord pedindo aos contribuidores para comentarem na tarefa;
  4. Espere o feedback do pessoal (24 horas já é o suficiente);
  5. Comece a desenvolver seu código: faça sua branch a partir da develop;
  6. Abra um PR linkando a issue e coloque (caso nao seja colocado automaticamente pelo GitHub) o Daniel Prado (@danieloprado) e o Fernando Moraes (@ffernandomoraes) como reviewers, juntamente com outros que você ache interessante olhar seu código;
  7. Mova a tarefa no Board;
  8. Quando a PR passar pela verificação automatica será disparado uma mensagem no Discord, avisando que uma PR está pronta para o review;
  9. Veja os comentários da PR e, se tudo estiver correto, ela será mergeada com a develop e estará na proximá versão.

Backlog#

Accesse o Board para ver as tarefas e os componentes que faltam.

Padrões#

Tente seguir ao máximo a sintaxe dos componentes e funcionalidades já existentes.

Lembre-se de cada responsabilidade:

PackageDescrição
@eduzz/houston-uiComponentes do houston para web
@eduzz/eslint-config-houstonConfiguração padrão para o eslint
@eduzz/houston-formsHooks para validação de formulário (livre de plataforma. ex: web ou app)
@eduzz/houston-hooksHooks de uso geral (livre de plataforma. ex: web ou app)
@eduzz/coreConfigurações compartilhadas entre os projetos, não deve ser usado diretamente

Sempre que desenvolver um componente ou qualquer funcionalidade, adicionar no projeto docs, presente no workspace, uma documentação com os detalhes do componente/funcionalidade: sua finalidade, como usá-lo, suas propriedades, configurações, etc.

Documentação#

Para criar uma documentação do novo componente ou funcionalidade, navegue até a pasta docs do workspace e no terminal execute o comando:

yarn houston-docs

O CLI irá perguntar qual tipo de documentação você quer criar: documentação de componente, hook ou outro. Selecione a opção desejada através das setas do teclado:

Informe qual id e título da sua documentação.

Ao finalizar, será criado um componente seguindo o template correspondente ao tipo de documentação que deseja criar. Você pode encontrar o arquivo da documentação no projeto em docs/[tipo_da_doc]/[id].mdx.

Toda o projeto da documentação foi desenvolvido utilizando o Docusaurus.

Importando componentes React na documentação#

Os arquivos de documentação utilizam a extensão .mdx, permitindo que você escreva JSX dentro de seus arquivos Markdown e renderizá-los como componentes React.

import MyComponent from './myComponent';
<MyComponent />;

Além disso, você pode exibir o código-fonte do seu componente utilizando !!raw-loader!:

import CodeBlock from '@theme/CodeBlock';
import MyComponentSource from '!!raw-loader!./myComponent';
<CodeBlock className="language-jsx">{MyComponentSource}</CodeBlock>;

Adicionando sua documentação no menu lateral#

Para que sua nova documentação apareça no menu lateral, é necessário editar o arquivo sidebars.js na raiz do projeto docs, inserindo o caminho do arquivo .mdx da sua documentação no menu correspondente:

module.exports = {
docs: {
'Comece a usar': [
'getting-started/installation',
'getting-started/contributing'
],
Componentes: [
'components/alert',
'components/checkbox'
],
Hooks: [
]
},
};

Por baixo dos panos#

Utilizamos o ultra-runner para gerenciar o monorepo. O ultra-runner é bem simples e todos os projetos já estão configurados com build, dev e precommit unificados, assim como todos os projetos utilizam a configuração do eslint padrão que disponibilizamos junto com o package @eduzz/eslint-config-houston.
